Visual Focus - Exercise

Visual Focus is a cue to indicate for the user which part of the application is currently active. Now it's time to apply what you learned in the introduction section. Try to first fix it on your own, and then compare your code with the solution.

MVP

Write instructions for the deverloper to fix the accessibility of the following form.Provide visual reference (mock screenshot or video).

  • Only one focus at a time.
  • All interactive elements should have a focused state (not just inputs, but buttons, links, etc)
  • It should be obvious what element is focused
  • It should be distinguishable from other state such as disabled or active
  • Focused radio inputs should be easily identifiable from selected ones
  • focused links/buttons should be different and easily indefinable from hovered or visited links/buttons
  • focus styles should avoid "upsetting" the layout (e.g. adding a 1px border can make an input 2px wider than it was before focus. This is fine as long as it doesn't break the page flow)
  • It should consider the accessibility - color and contrast should be inclusive of users with vision impairments. Color alone is probably not sufficient to show focus.
  • It should consider any variation of background a focused element may appear on (a double halo/border can be useful to target dark and light backgrounds)
  • It should consider that some browser modes may interfere with, or eliminate, some styles (dark modes, high-contrast modes etc)
  • It should work for both keyboard focus and mouse focus
  • Invisible elements should generally not be focusable (an exception would be a skip link, for example)
